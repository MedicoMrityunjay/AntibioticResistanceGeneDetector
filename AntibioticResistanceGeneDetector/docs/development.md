# Development Workflow & Contributing

## Branching
- Use `main` for stable development
- Feature branches for new features

## Tagging
- RC tags: `vX.Y.Z-rcN` (TestPyPI)
- Stable tags: `vX.Y.Z` (PyPI)

## CI/CD
- All pushes/PRs run tests and validation
- RC tags trigger TestPyPI publish
- Stable tags trigger PyPI publish

## Contributing
- Fork the repo, create a feature branch, submit PR
- Follow code style and add tests
